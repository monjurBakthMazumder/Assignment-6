Discuss the scope of var, let, and const

var:

var is function scope or global scope.If you declare a variable from any place using var,it can be accessed from any place.Declaring a variable using var can change the value of that variable and declare a variable with the same name repeatedly,thereby changing the value of the previously declared variable.Which causes a lot of problems.

let:

let is block scope.It's also hoisted,but it's not initialized until the code execution reaches its declaration.If you declare a variable using let,its value can be changed later but no other variable with the same name can be declared later.

const:

const is block scope.it's hoisted and not initialized until the declaration.If you declare a variable using Const,the value of that variable cannot be changed and no other variable can be declared with the same name.

Tell us the use cases of null and undefined

Undefined:

1.A variable is considered undefined when its value is undefined.

2.If the pattern parameters are given to the function but less parameters than the parameters of the function are given when calling the function then the remaining parameters are considered as undefined.

3.Attempting to access a value of objects that does not exist in objects. In that case it will be treated as undefined.

4.If nothing is returned in the function, it will be treated as undefined.

5.Attempting to access a value of an Array that is not in the Array. In that case it will be treated as undefined.

null:

1.Declaring a variable whose value is not specified.

2.Instead of declaring the value with zero.

3.To set the value of a variable to zero.

4.If the value of a variable is unknown.

5.Later, you will determine the value in this case.

What do you mean by REST API?

Rest api is an important service. Which is used in web development. The full application programming interface. In the meantime, the data is sent from one web application to another web application. Using api, we can access data remotely and communicate between servers and applications. We can get, post, delete, put and patch data using api such as:

1.Post: To send data to server or database.

2.Get: To load data from server or database.

3.Delete: To delete any data from the server or database.

4.Put: To update any data in the server or database, if the data is not there in the database then it will create a new one.

5.Patch: To update any data on the server or database.

Discuss the scope of var, let, and const

var:

var দিয়ে ঘোষণা করা veritable function scope.var দিয়ে একটি ভেরিয়েবল কোন ব্লক এর ভিতরে ঘোষণা করলে তা বাইর থেকে access করা যায়। var দিয়ে ঘোষণা করা ভেরিয়েবলের মান বার বার পরিবর্তন করা যায় এবং যদি পরবর্তীতে একই নামে অন্য কোন ভেরিয়েবল ঘোষণা করা হয় তখন পূর্বের ভেরিয়েবলের মান ও পরিবর্তন হয়ে যায়, যার ফলে অনেক সমস্যা হয়।

let:

let দিয়ে ঘোষণা করা ভেরিয়েবল block scope থাকে। এটি ব্লক এর বাহির থেকে access করা যায় না, let দিয়ে ঘোষণা করা ভেরিয়েবলের মান বার বার পরিবর্তন করা যায় কিন্তু একই নামে অন্য কোন ভেরিয়েবল ঘোষণা করা যায় না।

const:

const দিয়ে ঘোষণা করা ভেরিয়েবল block scope থাকে। এটি ব্লক এর বাহির থেকে access করা যায় না,const দিয়ে ঘোষণা করা ভেরিয়েবলের মান পরিবর্তন করা যায় না এবং একই নামে অন্য কোন ভেরিয়েবল ঘোষণা করা যায় না।

Tell us the use cases of null and undefined

Undefined:

যখন কোনও ভ্যারিয়েবলে মান বলে দেওয়া হয় না, তখন সেটি undefined দেখায়। কোন ভেরিয়েবলের মান না থকলে তার ডাটা টাইপ undefined দেখাবে। undefined কখন দেখায় যখন-

১. যখন কোন ভেরিয়েবলের মান নির্দিষ্ট নয় তখন তা undefined হিসেবে বিবেচিত হয়।

২. যদি function এ নিদর্শন parameters দেওয়া থাকে কিন্তু function কে call করার সময় function এর parameters এর তুলনায় কম parameters দেওয়া হয় তখন অবশিষ্ট parameters undefined হিসেবে বিবেচিত হয়।

৩. objects এর এমন কোন মান access করার চেষ্টা করা যা objects এ নেই। সেক্ষেত্রে undefined হিসেবে বিবেচিত হবে।

৪. যদি Function এ কোন কিছু return করা না হয় তখন তা undefined হিসেবে বিবেচিত হবে।

৫. Array এর এমন কোন মান access করার চেষ্টা করা যা Array তে নেই। সেক্ষেত্রে undefined হিসেবে বিবেচিত হবে।

২. শূন্য দিয়ে মান ঘোষণা করার পরিবর্তে।

৩. কোন ভেরিয়েবলের মান শূন্য করতে।

৪. কোন ভেরিয়েবলের মান অজানা থাকলে।

৫. পরবর্তীতে মান নির্ধারণ করবেন এ ক্ষেত্রে।

What do you mean by REST API?

Rest api এটি গুরুত্বপূর্ণ পরিষেবা। যা web development এ ব্যবহার করা হয়। এর পূর্ণরূপ Application programming interface. এর মধ্যমে ডাটা একটি ওয়েব application থেকে অন্য web application এ পাঠানো হয়। api ব্যবহার করে আমরা ডাটা রিমোটলি access করতে পারি এবং সার্ভার এবং application এর মধ্যে communication করতে পারি। আমরা api ব্যবহার করে ডাটা Get, Post, Delete, Put এবং patch করতে পারি যেমন-

1.Post: সার্ভার বা ডাটাবেইজ এ ডাটা পাঠাতে।

2.Get: সার্ভার বা ডাটাবেইজ থেকে ডাটা load করতে।

3.Delete: সার্ভার বা ডাটাবেইজ থেকে কোন ডাটা Delete করতে।

4.Put: সার্ভার বা ডাটাবেইজ এ কোন ডাটা Update করতে, যদি ডাটাবেইজ এ ওই ডাটা না থাকে তাহলে নতুন করে তৈরি করবে।

5.Patch: সার্ভার বা ডাটাবেইজ এ কোন ডাটা আছে সেটাকে Update করতে।
